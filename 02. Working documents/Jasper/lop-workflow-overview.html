v0.1 overview

Letter of Proposal (LoP) workflow in Cursor

A plain-language map of what lives in this repo, how a beach consultant would use it, and where to click next. Built to speed up a strong first draft with clear guardrails—not to replace your partner.

Start here (30 seconds)

  1. Open the project index. AGENTS.md at the repo root lists rules and links to the runbook and prompts.
  2. Follow the runbook. It walks through Step 0 (which files count for this LoP), intake, gates, chapters, coach, then PPT + HTML packaging.
  3. Copy prompts into chat with @ on your sources—see the prompt pack README for order.

What you actually “got”

Four layers of product, all in markdown except this page:

1. Project brain (Cursor rules)

Files under .cursor/rules/ tell every chat how to behave: spine (including Why McKinsey), input hygiene (Step 0 manifest, no invented disclaimers), tools and data sources, and peer-folder etiquette.

Your review: open lop-builder-workflow.mdc, lop-builder-core.mdc, lop-builder-tools-data.mdc in the repo.

2. Tracker template (per pursuit)

Copy into each opportunity folder as lop-build-tracker.md. It now includes a Step 0 source manifest (so inputs don’t get mixed across LoPs), HITL gates A/B/C, boilerplate row, pilot timing, and an updated spine row for Why McKinsey.

Fleur — lop-build-tracker.template.md

Also mirrored in: Jasper/templates/lop-pursuit-template/lop-build-tracker.md

3. Runbook (the “how to” script)

One document: ordered steps from manifest → intake → Gate A → chapter passes (with v0.1 policy: e.g. agent-first Why McKinsey / Team / Credentials; human-first Approach) → merge appendix → Gate B → LoP coach → fixes → assembler (PPT + HTML) → Gate C.

lop-cursor-runbook.md (this folder)

4. Prompt pack (copy-paste into Cursor)

Small files you paste into chat—one role at a time—with rules about sources, evidence, and TBDs. Covers the full spine (context, timeline/team, market, fees, appendix merge, etc.), plus coach and assembler.
