# Background material — how Sjors LoP agents use gold proposals

All **`sjors-lop-agent-*.md`** prompts assume you **`@`** attach **1–3** files from **`01. Background material/`** (PDF/PPTX/MD) for **pattern and tone**, unless you are only polishing wording inside an already-approved draft.

## Rules (non-negotiable)

1. **Structure and phrasing** — action-title rhythm, slide density, bullet parallelism, exhibit habits.
2. **No cross-client facts** — never copy client names, metrics, fees, timelines, or confidential lines from a gold pursuit into **this** LoP.
3. If the model cannot read PDF text well, paste **slide titles** or a short excerpt from the **workplan / why us / approach** sections of the gold file.

## What “good” looks like by chapter (quick cues)

| Chapter agent | Steal from gold |
|---------------|-----------------|
| Slide layouts (all chapters) | **Geometry from gold**: section label + action title + body blocks + source line — see **`slide-structures-from-background-material.md`**; use rule **`sjors-lop-agent-slide-structures.mdc`** when extracting layouts in Cursor |
| Context & objectives | How context slides group MECE objectives; RFP alignment phrasing |
| Why McKinsey | Proof ladder, evidence density, “so what” titles |
| Timeline & team | Neutral staffing language; milestone strip without internal jargon |
| Team | Card layout, role naming discipline |
| Credentials | Case selection pattern; outcome language without over-claiming |
| Market | Exhibit titles as insights; footnote discipline |
| Approach | Workshop / steerco storyline; hypothesis tone without fake precision |
| Fees | Layout for TBD vs sourced numbers — **never** copy fee numbers from gold |
| Appendix / refs | Ordering, CV checklist style |
| Workplan | Week-grid density; milestone row pattern |

## Repo-wide formatting

Client-facing output must obey **`.cursor/rules/mckinsey-document-standards.mdc`** (pyramid, action titles, palette/tables/sources conventions).
