Best Practice LOP document (retrieved in Step 0) — the quality anchor for the entire workflow. Before any intake or research begins, the agent identifies the industry & practice area from the RFP and retrieves the matching gold-standard LOP document from the Best Practice Library. This document defines what excellence looks like for each of the 11 chapters below, and is referenced at every stage: gap detection (S1), synthesis framing (S3), structure (S6), drafting (S7), and coaching (S8). These documents already exist; new ones will be created per practice area over time.

Step 0 — Practice Area Identification & Best Practice LOP Retrieval
Before any intake: identify industry/practice context → retrieve the gold-standard LOP document that anchors the entire workflow
AGENTIC — ANCHOR STEP
📥Data sources
📄 RFP / RFI (industry & practice signals) ⭐ Best Practice LOP Library ⚙️ Practice area taxonomy (config) ⚙️ Engagement type classifier (competitive / non-competitive / exploratory / relationship)
🔧Tools
🏷️ Practice area classifier 🔍 Best Practice LOP retriever 📋 Quality criteria extractor 📚 Best Practice Library (document store — documents exist; expanded over time)
⚡Transformation
1
Read RFP signals → classify industry (e.g. Financial Services, Operations, Technology) and practice area
2
Classify engagement type: competitive tender / non-competitive / exploratory / relationship-building — each requires a different LOP tone and emphasis
3
Query Best Practice LOP Library → retrieve the matching gold-standard document(s) for this practice + engagement type
4
Extract key quality criteria from the best practice doc: chapter depth expectations, evidence density, tone, win-theme structure
5
Package as quality benchmark — this becomes the persistent reference for Steps 1, 3, 6, 7, and 8
📤 Output
Identified practice area & engagement type ⭐ Best Practice LOP reference document Quality criteria checklist (per chapter) Persistent anchor → Steps 1, 3, 6, 7, 8
Practice area + Best Practice LOP reference

Step 1 — Intake & Capture
Ingest all available materials; map coverage against 11 LOP chapters
AGENTIC
📥Data sources
⭐ Best Practice LOP (Step 0) 📄 RFP / RFI document 📄 Objectives brief 📄 Team roster & CVs 📄 Fee schedule 📄 Prior engagement refs 📄 Existing CST context 📄 Appendix / exhibits
🔧Tools
📑 Doc parser (PDF/DOCX/PPTX) ✂️ Text chunker 🏷️ Chapter-theme tagger 📊 Gap analyzer 💾 Intake store
⚡Transformation
1
Parse binary files (PDF, DOCX, PPTX) → extract clean text
2
Chunk text into topic segments; preserve source metadata
3
Tag each chunk to one of the 11 LOP chapters
4
Score coverage per chapter vs Best Practice LOP expectations: complete / partial / missing / below standard
5
Emit ranked gap list — chapters needing research or partner input to meet best practice depth
📤 Output
Structured intake package Chapter gap list (benchmarked against Best Practice LOP) Flows to → Step 2
⭐ Best Practice LOP active — gap scoring calibrated against practice-area quality expectations
Intake package + chapter gap list
🤖
Step 2 — Build Context
Fill chapter gaps with external research; build client & market picture
AGENTIC
📥Data sources
🔗 Chapter gap list (Step 1) 🌐 Web (client site, news, analyst reports) 🌐 Market & competitor public data 📚 McKinsey Know / KB 📚 MVI (McKinsey Value Intelligence) 📄 CST prior work notes
🔧Tools
🌐 Web search connector 📚 McKinsey Know API 📚 MVI connector (stub v1) 🔍 Research synthesis LLM 📎 Citation tracker
⚡Transformation
1
For each gap chapter → formulate targeted search queries
2
Retrieve from web + McKinsey KB; de-duplicate + rank by relevance
3
Synthesize per chapter; attach source citations
4
Chapters enriched: Context & Objectives · Market Trends · Why McKinsey · Prior Examples · Credentials
5
Flag residual evidence gaps where retrieval returned nothing useful
📤 Output
Enriched context doc (per chapter) Source citation index Residual evidence-gap flags Flows to → Step 3
Enriched context + citations
🤖
Step 3 — Synthesis
Merge everything known; produce problem statement and targeted partner question list
AGENTIC
📥Data sources
⭐ Best Practice LOP (Step 0) 🔗 Structured intake (Step 1) 🔗 Enriched context (Step 2) 🔗 Chapter gap list (Step 1)
🔧Tools
🧠 Synthesis LLM 📝 Problem statement framer ❓ Question list generator
⚡Transformation
1
Merge intake + research into single coherent picture
2
Identify key themes, tensions, and must-win angles — using Best Practice LOP to calibrate what a winning narrative looks like for this practice area
3
Sharpen problem statement; frame Context & Objectives chapter
4
Generate partner question list — only questions unanswerable from docs; designed for ~10-min voice answer
5
Flag any conflicting inputs for human review at Gate A
📤 Output
Brief synthesis doc Sharpened problem statement ~10-min partner question list Flows to → Gate A
⭐ Best Practice LOP active — narrative calibrated against practice-area win-theme patterns
Synthesis doc + question list

HITL GATE
👤 Human role
Read synthesis + problem statement. Verify accuracy and priorities. Check question list covers all chapter gaps — especially Approach, Fees, Why McKinsey, Team. Flag any conflicting inputs before dispatching to partner.

✓ ApproveDispatch question list to partner → proceed to Step 4
↩ ReviseReturn to Step 3 with specific correction notes

Step 5 — Validate Answers & Flag Gaps
Cross-reference answers against all chapter gaps; score completeness
AGENTIC
📥Data sources
🔗 Transcribed answers (Step 4) 🔗 Chapter gap list (Step 1) 🔗 Question checklist (Step 3) 🔗 Intake package (Step 1)
🔧Tools
✅ Validation agent 📊 Completeness scorer ⚠️ Contradiction detector
⚡Transformation
1
Match each answer to its target chapter gap item
2
Check for contradictions with intake documents
3
Flag answers that are incomplete, vague, or ambiguous
4
Score residual gaps by chapter: severity (blocks structure / acceptable unknown)
📤 Output
Validated answers Residual chapter gap report (severity-ranked) Flows to → Gate B
Validated answers + gap report
🚦 Gate B — Human Decision: Resolve chapter gaps before structuring
HITL GATE
👤 Human role
Review gap report. For each flagged chapter decide: provide additional information, accept as known unknown (will be flagged in the LOP), or escalate to partner for follow-up. Nothing that blocks the Approach or Fees chapter can pass through unresolved.
↩️ Decision paths
✓ ResolvedGaps filled or formally accepted → proceed to Step 6
📋 Accept gapMark as known unknown in LOP → proceed to Step 6 with flag
↩ More infoHuman provides additional context → re-run Step 5 validation
Gaps resolved / accepted

Step 6 — Dot-Dash Structure
Map all validated content to 11 chapters; generate page-level outlines with strong descriptions
AGENTIC + optional human steering
📥Data sources
⭐ Best Practice LOP (Step 0) 🔗 Validated answers (Step 5) 🔗 Problem statement (Step 3) 🔗 Enriched context (Step 2) ⚙️ Per-chapter guidelines (config) ⚙️ Stakeholder map ⚙️ Approved sources list
🔧Tools
🏗️ Structure agent 📋 Guidelines loader 🔗 Chapter dependency resolver
⚡Transformation
1
Map all validated content to the 11 LOP chapters
2
Use Best Practice LOP chapter structure as the template — mirror its chapter depth, page count, and evidence density for this practice area
3
Apply per-chapter guidelines; resolve chapter ordering dependencies
4
Generate dot-dash outline: chapter → pages → per-page strong descriptions
5
Human may steer outline before handoff to drafters (optional)
📤 Output
Dot-dash structure doc (11 chapters → N pages → per-page descriptions) Flows to → Step 7
⭐ Best Practice LOP active — chapter structure and page depth modelled on practice-area gold standard
Dot-dash structure → chapter briefs

Step 7 — Draft Pages
Chapter drafter agents write each of the 11 chapters in parallel
AGENTIC — parallelised
📥Data sources
⭐ Best Practice LOP (Step 0) 🔗 Dot-dash structure (Step 6) 🔗 Full research pack (Steps 1 + 2) 🔗 Validated answers (Step 5) ⚙️ Approved sources list ⚙️ Style & tone guidance
🔧Tools
✍️ Chapter drafter agents (×11) 🔗 Evidence retriever 📎 Citation inserter 🚩 Evidence-needed tagger
⚡Transformation
1
Each drafter receives its chapter's dot-dash brief and source bundle
2
Write page-by-page to the dot-dash description; follow style guide; use Best Practice LOP as the writing quality target for tone, argument density, and evidence use
3
Retrieve and insert supporting evidence from research pack
4
Tag [EVIDENCE NEEDED] placeholders where sourcing is missing
5
Chapters run in parallel where dependency DAG allows
📤 Output
Full draft — all 11 chapters Evidence-needed placeholders Flows to → Step 8
⭐ Best Practice LOP active — chapter writing quality benchmarked against practice-area gold standard
Page drafts + evidence placeholders
🤖
Step 8 — LOP Coach — Critical Analysis
Score draft for quality, unsupported claims, and gaps; produce must-fix list
AGENTIC
📥Data sources
⭐ Best Practice LOP (Step 0) — primary scoring benchmark 🔗 Full draft (Step 7) 🔗 Source citation index (Step 2) ⚙️ LOP quality rubric
🔧Tools
🔍 LOP coach agent 📊 Quality rubric engine ⚠️ Claim validator 📋 Chapter coherence checker
⚡Transformation
1
Analyze each page against LOP quality rubric AND the Best Practice LOP for this practice area
2
Score chapter-by-chapter quality delta vs best practice: argument strength, evidence density, win-theme clarity
3
Identify unsupported claims — flag any assertion without a source citation
4
Check chapter coherence — does the narrative flow logically across all 11 chapters?
5
Categorise issues: must-fix (blocks send) vs nice-to-have; cite specific best practice gap where relevant
📤 Output
Quality score per chapter vs Best Practice LOP Must-fix issues list Nice-to-have improvements Flows to → Gate C
⭐ Best Practice LOP active — quality delta scored per chapter against practice-area gold standard
Scored issues list
🚦 Gate C — Human Decision: Approve draft for packaging & external send
HITL GATE
👤 Human role
Review coach report and full draft. Verify all must-fix issues are addressed. No external send without explicit human approval. Optionally provide revision instructions chapter-by-chapter.
↩️ Decision paths
✓ ApprovedProceed to Step 9 (assemble + dispatch)
↩ Loop backProvide revision comments → Step 10 applies fixes; loops to Step 7 or 8
Approved
🤖

Dispatch to reviewer list via email connector

PPT / HTML artifact Dispatch confirmation Flows to → Step 10
PPT / HTML artifact to reviewers

Step 10 — Capture Comments & Iterate
Human prioritises reviewer comments; system applies only approved fixes
HUMAN + SYSTEM
📥Data sources
💬 Reviewer comments 🔗 Current draft (Step 7 / 9) 🔗 Coach report (Step 8)
🔧Tools
📝 Comment parser ✏️ Revision agent 📋 Change log
⚡Transformation
1
HUMAN: Provide reviewer comments; tag as must-fix vs optional per chapter
2
SYSTEM: Parse comments → group by chapter → apply only human-approved fixes
3
Log all changes to prevent infinite rewrite loops
4
Route back: revise pages → Step 7 · re-coach only → Step 8
📤 Output
Revised draft Change log Loops to → Step 7 (revise pages) or Step 8 (re-coach)
